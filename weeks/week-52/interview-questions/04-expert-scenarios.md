## Q101: Portfolio Panel Defense

| Attribute | Value |
|-----------|-------|
| **Difficulty** | Expert |
| **Category** | Graduation Capstone |
| **Frequency** | Very Common |
| **Type** | Scenario |

### Question

In graduation panel, how defend portfolio case study under skeptical technical review?

### Short Answer (30 seconds)

Walk trade-offs first, cite NFRs, admit what you'd change, link to principles. Invite deep dive on riskiest decision.

### Detailed Answer

**Defense framework:**
1. Restate constraints and NFRs
2. Present recommended option + rejected alternatives
3. Quantify outcome
4. 'What I'd change today' — growth signal
5. Offer whiteboard extension

**Skeptic question:** 'Why not serverless?' — answer with constraints not defensiveness.

### Architecture Perspective

Portfolio defense simulates hiring loop — capstone finale.

### Follow-up Questions

1. **Don't know detail? — Honest 'I'd revisit ADR-12' better than bluff.**
2. **Time limit? — 7 min case + 3 min Q&A rehearsed.**

### Common Mistakes in Interviews

- Read slides verbatim
- No trade-offs mentioned
- Bluff on unfamiliar probe

### Detailed Answer

**4-hour simulation:**

2. Break 15m
3. System design 60m

7. Leadership 60m

**Energy:** Hardest mock slot when tired (session 3). Notes sheet allowed?

### Architecture Perspective
