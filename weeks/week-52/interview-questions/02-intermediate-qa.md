## Q031: Portfolio Case Study Selection

| Attribute | Value |
|-----------|-------|
| **Difficulty** | Intermediate |
| **Category** | Graduation Capstone |
| **Frequency** | Very Common |

### Question

How select 3–5 case studies for architecture portfolio?

### Short Answer (30 seconds)

Depth over breadth — pick programs with constraints, trade-offs, metrics, and clear 'I' role.

### Detailed Answer (3–5 minutes)

**Selection matrix:**
| Criterion | Weight |
|-----------|--------|
| Your explicit role | High |
| Quantified outcome | High |
| Technical depth | High |
| Confidentiality safe | Required |
| Variety (scale, domain) | Medium |

**Ideal mix:** 1 platform, 1 migration, 1 greenfield or crisis.

**Reject:** Resume bullet expansion; group project without your role.

### Architecture Perspective

Case study selection is curatorial — interviewers deep-dive one story.

### Follow-up Questions

1. **How old? — Last 5 years preferred; timeless patterns OK.**
2. **NDA? — Anonymize; generic diagrams fine.**

### Common Mistakes in Interviews

- Seven shallow cases
- No metrics
- Confidential customer names

## Q032: Portfolio PDF vs Website

| Attribute | Value |
|-----------|-------|
| **Difficulty** | Intermediate |
| **Category** | Graduation Capstone |
| **Frequency** | Very Common |

### Question

Portfolio PDF or static website — which format for architect job search?

### Short Answer (30 seconds)

PDF for recruiters and application systems; optional GitHub Pages site for discoverability. Same content, two channels.

### Detailed Answer (3–5 minutes)

**PDF:**
- Attached to applications
- Controlled layout
- 15–25 pages max

**Website:**
- SEO and inbound
- Featured talks/blogs
- Custom domain optional

**Both:** Sync quarterly; canonical PDF version numbered.

**Accessibility:** PDF tagged; site mobile-friendly.

### Architecture Perspective

Format choice is distribution strategy — content quality is constant.

### Follow-up Questions

1. **Password protect? — OK for semi-confidential; provide on request.**
2. **Interactive demos? — Bonus for platform architects.**

### Common Mistakes in Interviews

- Only LinkedIn profile
- 100-page unedited dump
- Broken links on site

## Q033: 4-Week Mock Schedule

| Attribute | Value |
|-----------|-------|
| **Difficulty** | Intermediate |
| **Category** | Graduation Capstone |
| **Frequency** | Very Common |

### Question

Detail 4-week mock interview schedule before architect job search.

### Short Answer (30 seconds)

Week 1 system design, week 2 behavioral, week 3 leadership, week 4 full loop — deliberate practice with debrief log.

### Detailed Answer (3–5 minutes)

**Schedule:**
| Week | Sessions | Focus |
|------|----------|-------|
| 1 | 3×90min | URL shortener, payments, SaaS |
| 2 | 2×60min | STAR recording review |
| 3 | 2×60min | Exec comms, conflict |
| 4 | 1×4hr | Simulated loop + debrief |

**Rules:** Max 2 mocks/day; always debrief; gap spreadsheet.

### Architecture Perspective

Scheduled mocks beat ad-hoc cramming.

### Follow-up Questions

1. **Paid coach? — Final week polish only if budget allows.**
2. **Peer matching? — Reciprocal mocks with architect peer.**

### Common Mistakes in Interviews

- Only reading books
- No recording review
- Skip debrief

## Q034: Mock Debrief Rubric

| Attribute | Value |
|-----------|-------|
| **Difficulty** | Intermediate |
| **Category** | Graduation Capstone |
| **Frequency** | Very Common |

### Question

What rubric debrief each architecture mock interview?

### Short Answer (30 seconds)

Score requirements, estimation, diagram, NFRs, trade-offs, communication — 1–5 each; track trends.

### Detailed Answer (3–5 minutes)

**Rubric dimensions:**
1. Requirements clarification
2. Estimation sanity
3. API/data model clarity
4. Scale path articulated
5. NFR coverage (DR, security, ops)
6. Trade-off explicitness
7. Communication/time mgmt

**Log:** Date, problem, scores, top 2 gaps, drill plan.

### Architecture Perspective

Rubric turns mocks into measurable improvement.

### Follow-up Questions

1. **Behavioral rubric? — STAR clarity, time, probe handling.**
2. **Trend over 4 weeks? — Scores should rise on weak dims.**

### Common Mistakes in Interviews

- Vague 'went OK' debrief
- Same weakness unaddressed
- No time-box in mocks
